安装包内容：

  • jdk1.8.0_131: 自带JDK，默认安装包内所有项目使用该JDK

  • sup_apache-tomcat-8.5.9: Tomcat（已安装MOF探针），其webapps包含apphub.war（AppHub UAV前端JEE程序），rocket.war（RocketMQ监控支持）

  • sup_elasticsearch: ES 5.4.1

  • sup_mongodb: MongoDB

  • sup_redis: Redis

  • sup_rocketmq: RockeMQ

  • uav: UAV.Monitor+APM根目录

  • uavagent: 监控代理程序（Agent）已安装MOF探针
  • uavhm: 健康管理服务（HM）已安装MOF探针
  • uavmof: MOF探针

脚本说明：

  • start_all.bat: 启动并后台运行所有（第三方服务、UAV服务、AppHub），然后打开http://localhost:8080/apphub

  • start_apphub.bat: 启动包含AppHub的Tomcat

  • start_monitor_agent.bat: 启动监控代理程序

  • start_monitor_healthmanager.bat: 启动健康管理程序

  • start_thirdparty_support.bat: 启动并后台运行第三方支持

  • reset_all.bat: 清除第三方和UAV服务的数据信息，恢复安装包刚解压的状态

问题说明:

1.如果遇到不能看到任何应用（上帝之眼等）的情况，说明权限数据没有进入MongoDB

  • 请分别检查Redis，MongoDB，RocketMQ，ElasticSearch是否都已经正常启动，并检查是否由于Windows权限或本机安装的防火墙阻止了其端口运行

  • 关闭已经打开的所有程序，并尝试执行以下操作

1)执行reset_all.bat或手工删除以下目录的文件（不要直接删除目录，只删除内容）

sup_elasticsearch\data\*
sup_mongodb\Server\data\*
sup_redis\dump.rdb

2)按以下顺序在新终端中分别启动, 再观察Redis，MongoDB，RocketMQ，ElasticSearch是否都已经正常启动

start_thirdparty_support.bat
start_monitor_healthmanager.bat
start_monitor_agent.bat
start_apphub.bat
